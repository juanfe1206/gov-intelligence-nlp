## Pattern Analysis Across Story Records

### Common Struggles

- **Date/time correctness** appeared repeatedly (default window boundaries, UTC/local date handling, spike window interpretation).
- **Data filtering correctness** repeated in backend reviews (NULL-safe `error_status`, param propagation completeness, minimum comparison party rules).
- **UI state consistency** repeated in frontend reviews (empty vs main branch parity, neutral precondition messaging, feedback on failed actions).

### Recurring Review Themes

- Guardrails for edge-case inputs needed reinforcement:
  - invalid or incomplete query combinations
  - empty or partial comparison setup
  - encoding and transport details for URL-based actions
- Copy/export workflows needed explicit user feedback and complete context serialization (notably selected parties and topic labels).

### Breakthrough Moments

- The analytics service became a reusable core: later stories successfully composed existing service functions instead of duplicating query logic.
- A stable pattern for frontend async fetch safety (`AbortController` + active-flag guards) reduced race-condition style bugs in panel components.
- Review feedback loops became fast and pragmatic, yielding measurable hardening without stopping delivery flow.
