## What went well

- **Contract-first connector architecture:** Story 5.1 established clear abstractions (`BaseConnector`, `NormalizedPost`, validation summary), which reduced ambiguity in later stories.
- **Incremental delivery strategy worked:** Epic progressed from interface -> first connector -> replay mode -> observability/retry -> compliance guardrails without major rework.
- **Strong review feedback loops:** Story files show repeated patch/defer tracking that prevented silent regressions and improved implementation quality.
- **Operational depth improved:** Checkpointing, replay mode, failure taxonomy, and `/jobs` visibility provide practical diagnostics for connector runs.
- **Compliance posture became explicit:** `.env.example`, connector metadata YAML, and log-safety checks moved safety from implicit conventions to documented policy.

## Challenges and friction

- **Early status inconsistency:** Story `5.1` file still shows `Status: review` while sprint tracker marks it `done`. This is a process artifact mismatch.
- **Connector service complexity growth:** `service.py` absorbed retry behavior, job persistence, checkpoint policy, and error categorization; risk of becoming a hotspot.
- **Deferred technical concerns accumulated:** Session/transaction boundaries, async blocking fetch path, and checkpoint edge behavior were intentionally deferred to keep scope moving.
- **Testing split between unit and DB-required coverage:** Some observability and integration checks depend on DB-enabled test runs, creating uneven confidence if DB tests are skipped.

## Pattern analysis across story records

### Common struggles (2+ stories)

- Correctness around **checkpoint semantics** (`> vs >=`, replay should not advance cursor).
- **Dedup behavior** at multiple layers (content hash + partial unique index + integrity handling).
- **Job metadata consistency** (`mode`, `normalized_count`, `failure_category`) across model, API schema, and persistence.
- **Security-by-default hygiene** (no secret logging, safe defaults, documented constraints).

### Recurring review themes

- Resolve patch-level issues quickly, defer only clearly scoped non-blockers.
- Keep API contracts and job schema synchronized when evolving connector features.
- Prefer explicit, auditable behavior over implicit assumptions for run summaries and operational limits.

### Breakthrough moments

- **5.2:** First end-to-end offline connector with checkpoint table established a reusable pattern.
- **5.3:** Replay mode achieved deterministic behavior while preserving live checkpoint integrity.
- **5.4:** Error taxonomy plus bounded retries made failures diagnosable and machine-readable.
- **5.5:** Compliance metadata and max-record guardrails reduced operational and policy risk.
